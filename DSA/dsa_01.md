# What is DSA:
1. Data Structures Algorithms.
2. Yeh system mein data ko organize aur store karne ka tareeqa hai taake usay zaroorat parne par aasan aur jaldi se use kiya ja sakay.

# Algorithms:

1. **Algorithms**: Yeh kisi bhi kaam ko karne ka liya ya kisi problem ko solve karne ka liya aik step-by-step formula hota hai jo phela se pre-defined hota.

## Real-World Exmaple:

Chai banane ka tareeqa (algorithm) pehle se tay hai:

1. Paani ubaalein.
2. Patti aur cheeni dalein.
3. Doodh dalein.

Yeh steps **pehle se defined** hain, yeh badal nahi rahe. Lekin **input** ke hisaab se iska behavior badlega:

* Agar aap input mein **2 cup** paani/doodh denge, to algorithm 2 cup chai banayega.
* Agar aap input mein **100 cup** paani/doodh denge, to algorithm wahi rahega, lekin ab usay chalne mein zyaada time lagega (bartan bara chahiye hoga, memory/space zyaada lagegi).

Lekin aaj kal ki dunya mein aik aisi field bhi hai jahan algorithms input dekh kar apne aap ko khud badalte aur behtar karte hain—usay hum **Machine Learning (AI)** kehte hain. Wahan algorithm ka bunyadi dacha fixed hota hai, lekin wo data (input) se seekh kar apne rules ko khud update karta reitah hai.

# Purpose of DSA:

1. DSA ka Main Purpose sirf code likhna nahi hai, balki Code ki Efficiency (performance) aur Scalability ko behtar banana hai.

2. **Efficiency**: Aap ka code kitna fast chalta hai aur computer ki kitni memory (RAM) use karta hai.

3. **Scalability**: Jab aap ki website par sirf 10 users hon to har code fast chalta hai. Lekin jab 10,000 ya 1 million users aa jayein, to ghalat tarah se likha hua code website ko crash kar deta hai. DSA aap ko sikhata hai ke aam code aur scalable code mein kia farq hota hai.
